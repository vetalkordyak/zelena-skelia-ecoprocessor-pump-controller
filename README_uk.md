*(English version of this README: [`README.md`](./README.md).)*

Контролер циркуляційного насоса біофільтра на базі ESP8266 (Wemos D1 mini) з
плавним пульсуючим керуванням потужністю через AC-димер (фазове різання,
leading-edge). Прошивка є копією / аналогом прошивки контролера **EcoProcessor**
і повністю сумісна за принципом роботи з біосептиками **«Зелена Скеля»** —
той самий алгоритм циклічного зрошення (розгін → робота → спад → пауза),
багатоступеневий захист насоса від перевантажень та автоматичний
перезапуск після аварії.

### Біофільтр «Air Ring»® / «Eco Ring»

Пластикові шайби **«Air Ring»®** (або **«Eco Ring»**) — найважливіший
елемент біофільтра автономної каналізації. Виготовлені з твердого
пластику, корисна площа завантаження — **180 м² на 1 м³**.

- **Створення біоплівки.** Головне завдання шайб — забезпечити
  оптимальне середовище для прикріплення й розвитку корисних бактерій,
  які природним шляхом містяться у стоках. Мікроорганізми закріплюються
  на поверхні пластику, утворюючи біоплівку, і харчуються органічними
  забрудненнями стічних вод, окислюючи та розкладаючи їх.
- **Здатність до самоочищення.** Шайби самостійно очищаються від
  залишків відмерлої біоплівки. У старіших поколіннях фільтрів саме
  відмерлі бактерії викликали швидке замулення наповнювача, погіршення
  аерації й загнивання, що повністю зупиняло очищення. «Air Ring»® /
  «Eco Ring» не потребують спеціальної чистки чи заміни — вони довговічні
  й не руйнуються під час експлуатації.
- **Переваги над йоржовими та сотовими наповнювачами:** ефективність
  (більший контакт повітря/води/мікроорганізмів), екологічність (не
  виділяють дрібних частинок у воду чи ґрунт), економічність (менший
  тиск повітря без постійної роботи компресорів), практичність (успішно
  протестовані у понад 40 000 біофільтрів у понад 30 країнах світу — від
  спекотного клімату до Арктики).

### Роль циркуляційного насоса

1. **Забір освітленої води.** Спочатку побутові стічні води проходять
   ретельне механічне очищення від жирів, нерозчинних речовин та
   твердого бруду в трьох секціях септика.
2. **Підйом та зрошення (аерація).** Насос піднімає вже попередньо
   освітлену воду нагору — у біологічний реактор. Там він декілька разів
   зрошує (розпилює) рідину над шайбами «Air Ring»®. Завдяки цьому вода
   насичується киснем та активно взаємодіє з біоплівкою на шайбах, де
   бактерії завершують біологічний розклад органіки.

Використовується надійне насосне обладнання відомих світових брендів —
зокрема **Wilo, Kärcher або Grundfos**.

> ⚙️ **Max Power і Min Power підбираються вручну під конкретний септик.**
> Однакових налаштувань "на всі випадки" не існує — форсунки/розпилювачі,
> висота підйому та кількість шайб «Air Ring»® відрізняються від об'єкта
> до об'єкта. Тому після монтажу потрібно **виставити ці два параметри
> дослідно**, спостерігаючи за реальним зрошенням у біореакторі: рідина
> має **рівномірно покривати й зрошувати всі шайби**, без сухих
> ділянок (замало потужності — не долітає/не покриває) і без
> перехльостування чи розбризкування повз завантаження (забагато
> потужності). Ramp Up/Down тоді плавно "гойдає" потік між цими двома
> перевіреними межами, а не між випадковими значеннями.

### Роль контролера

Завдяки роботі інтелектуального блоку керування **EcoProcessor**, насос
захищений від перевантажень завдяки системі **багатоступеневого
захисту**. Крім того, контролер оптимізує цикли роботи насоса, знижуючи
загальне енергоспоживання системи **більш ніж у 10 разів**. Наприклад,
насос марки Wilo у такій системі споживає всього **33 Вт електроенергії
на годину**.

Ця ESPHome-прошивка реалізує ту саму ідею відкритим способом:

- **Циклічне зрошення замість безперервної роботи** (режими P1/P2, робота
  ⇄ пауза) — саме та оптимізація циклів, яка й дає багаторазову економію
  енергії.
- **Плавний фазовий розгін/спад потужності** (Ramp Up/Down) замість
  різкого вмикання на повну — м'якший старт і зупинка при кожному циклі
  зрошення, менше навантаження на насос.
- **Контроль струму насоса та автоматичний перезапуск** — контролер
  стежить за струмом і реагує на позаштатні режими роботи (перевантаження,
  заклинювання ротора), вимикаючи насос і пробуючи перезапустити його
  через паузу; власник бачить лічильник спроб (`Restart Attempts`) у
  веб-інтерфейсі.

## 2. Схема підключення (ASCII)

### 2.1 Силове коло 220В (мережа → димер → давач струму → насос)

ACS712 стоїть **на виході димера** (після фазового різання), у розриві
фазного проводу між димером і насосом — тобто вимірює саме той струм,
який реально йде на насос, а не загальний струм мережі до димера.

```
  ~220V AC MAINS
   L ●───────────────────────────────────────┐
   N ●───────────────────┐                   │
                          │                   │
                          ▼                   ▼
              ┌─────────────────────────────────────────┐
              │            AC Dimmer RobotDyn             │
              │                                           │
              │   AC-IN(N)                    AC-IN(L)    │
              │                                           │
              │            (triac, фазове різання)        │
              │                                           │
              │   AC-OUT(N)                  AC-OUT(L)    │
              └───────┬───────────────────────────┬───────┘
                      │                            │
                 GATE ●│                  Z-CROSS ●│   ── сигнали на ESP8266 (див. 2.2)
                      │                            │
                      │                            ▼
                      │                  ┌───────────────────┐
                      │                  │     ACS712-5A       │
                      │                  │   IP+ ────── IP−    │  ── давач струму,
                      │                  └──────┬───────┬──────┘     У РОЗРИВІ L,
                      │                     OUT │(analog)             ПІСЛЯ димера
                      │                         │       │
                      ▼                         │       ▼
              ┌───────────────────────────────────────────┐
              │                  НАСОС (Pump)               │
              │            N                    L           │
              └───────────────────────────────────────────┘
```

### 2.2 Керуючі сигнали → Wemos D1 mini (ESP8266)

```
  GATE (з димера)      ───────────►  GPIO4 (D2)
  Z-CROSS (з димера)   ◄───────────  GPIO5 (D1)
  ACS712 OUT (analog)  ───────────►  A0
  LED індикатор        ◄───────────  GPIO2 (D4, активний LOW)
```

### 2.3 Живлення логіки (220В → 5В, ізольовано)

HLK-5M05 подає 5V/GND одразу на **два** споживачі: Wemos D1 mini (через
VBUS/5V пін) і логічну сторону ACS712 (пін VCC/GND самого модуля — не
плутати з силовими клемами IP+/IP−, які стоять окремо в розриві L, див. 2.1):

```
                               ┌──► 5V/GND ──► Wemos D1 mini (VBUS, GND)
~220V AC MAINS ──► HLK-5M05 ──┤
   (AC-DC, ізольований)        └──► 5V/GND ──► ACS712 (VCC, GND — логічна сторона)
```

**Мапа піни ESP8266 (за кодом `pump_controller.yaml`):**

| Пін Wemos | Функція | Куди підключається |
|-----------|---------|---------------------|
| `GPIO4` (D2) | `gate_pin` димера | DIM на AC Dimmer RobotDyn |
| `GPIO5` (D1) | `zero_cross_pin` (вхід) | Z-C на AC Dimmer RobotDyn |
| `GPIO2` (D4) | вихід на світлодіод (інвертовано — активний LOW) | LED індикатор стану |
| `A0` | аналоговий вхід АЦП | OUT датчика струму ACS712 (логічна сторона) |
| `3V3` | живлення логіки димера | VCC на AC Dimmer RobotDyn |
| `5V`, `GND` (VBUS) | живлення | вихід HLK-5M05 (спільний з VCC/GND ACS712) |

> ⚠️ **Небезпека ураження струмом.** AC Dimmer, ACS712 (у розриві фази) та
> HLK-5M05 працюють під напругою мережі 220В. Усі роботи з монтажу — тільки
> при повністю знеструмленому щитку/лінії, бажано через автомат і УЗО.
> Wemos D1 mini з боку логіки (GPIO/A0/5V) гальванічно розв'язаний від мережі
> лише завдяки ізольованому блоку живлення HLK-5M05 — не використовуйте
> неізольовані блоки живлення.

## 3. Принцип роботи

Прошивка реалізує неблокуючий кінцевий автомат (`interval:` кожні 100мс) з
чотирма станами:

```
      0 = stopped
        │ (Pump: Enabled = ON)
        ▼
      1 = running ──(робота вийшла за Work Time)──► 2 = resting
        ▲                                                │
        │◄───────────────(вийшов Rest Time)──────────────┘
        │
        │ (аварія: немає струму при активному димері)
        ▼
      3 = fault ──(вийшов Retry Delay)──► назад у 1 = running
```

### Режими роботи (`select: Operating Mode`)

Перемикаються одним селектором у веб-інтерфейсі (група "Pump Control"),
без перепрошивання. При зміні режиму на льоту система одразу починає
новий цикл з фази "робота" (`g_state = 1`, ramp up).

#### P1 Standard — звичайний робочий режим

- Робота: **120с (2 хв)**, пауза: **480с (8 хв)** → повний цикл 10 хв,
  насос активний ≈ **20%** часу.
- Використання: типове щоденне навантаження — стоки надходять регулярно,
  потрібна регулярна циркуляція/відкачка без зайвого зносу насоса.
- Усі 4 часових параметри (Work/Rest/Ramp Up/Ramp Down Time) редагуються
  окремо в групі "Mode P1 — Standard".

#### P2 Economy — економний режим

- Робота: **60с (1 хв)**, пауза: **1740с (29 хв)** → повний цикл 30 хв,
  насос активний лише ≈ **3.3%** часу.
- Використання: періоди малого навантаження (ніч, відсутність мешканців,
  міжсезоння) — суттєво менше напрацювання насоса й споживання
  електроенергії, але рідина все одно не застоюється.
- Параметри повністю незалежні від P1 (окрема група "Mode P2 — Economy") —
  можна тримати обидва профілі готовими і перемикатись без перерахунку.

#### P3 Constant — режим тестування насоса

- Насос працює на **100% потужності безперервно**, без циклів "робота/
  пауза" і без пульсації рівня — рампи (Ramp Up/Down) тут не задіяні
  взагалі, димер фактично повністю відкритий.
- Використання: **виключно для тестування насоса на повній потужності
  без димера** — перевірити, чи насос сам по собі справний і видає
  нормальний струм/потік, без впливу фазового регулювання. Це не
  штатний робочий режим для біофільтра.
- Захист від аварії працює й тут, але окремою гілкою коду: перевіряється
  напряму, чи є струм при 100% потужності, без прив'язки до фази ramp
  up/down (яких у цьому режимі просто немає).

#### Пульсація потужності всередині фази "робота" (P1/P2)

На відміну від простого "увімкнув на фіксовану потужність", у режимах
P1/P2 рівень потужності всередині фази роботи безперервно **пульсує
трикутником** між **Min Power** і **Max Power**, поки не закінчиться час
роботи:

```
рівень (%)
 Max ─┤      ╱╲          ╱╲          ╱╲
      │     ╱  ╲        ╱  ╲        ╱  ╲
      │    ╱    ╲      ╱    ╲      ╱    ╲
 Min ─┤   ╱      ╲    ╱      ╲    ╱      ╲
      └──┴────────┴──┴────────┴──┴────────┴──► час
         Ramp Up  Ramp Down  Ramp Up  Ramp Down …
         (напр. 0.1с)  (напр. 14с)
```

Тобто один "зубець" = Ramp Up Time (швидкий розгін до Max) + Ramp Down
Time (повільніший спад до Min), і так по колу до кінця фази роботи.
За замовчуванням розгін значно швидший за спад (0.1с проти 14с) — майже
миттєвий "поштовх" і плавне згасання, а не симетрична синусоїда. Це і є
та сама "плавна пульсація", згадана на початку README — вона м'якша для
насоса й труб, ніж утримання одного фіксованого рівня потужності всю
фазу роботи.

#### Приклад заводських налаштувань (з відео)

На офіційному відео «Зелена Скеля» (джерело — див. початок README)
виробник наводить приклад налаштування блоку керування EcoProcessor для
трьох моделей септика. Оригінальний запис (мовою відео) і його розшифровка
в термінах цієї прошивки — поруч, для перевірки:

**Позначення на відео:** `H` = Пауза (Rest, хв), `H.` = Робота (Work, хв),
`h` = Спад (Ramp Down, с), `h.` = Розгін (Ramp Up, с), `V` = Min Power (%),
`V.` = Max Power (%).

| Модель | Режим | Оригінал з відео | Work Time | Rest Time | Ramp Up | Ramp Down | Min Power | Max Power |
|--------|-------|-------------------|-----------|-----------|---------|-----------|-----------|-----------|
| **ZS4** | P1 (Р-1) | `H8 H.2 \ h14 h.1 \ V32 V.38` | 120с (2 хв) | 480с (8 хв) | 0.1с | 14с | 32% | 38% |
| **ZS4** | P2 (Р-2) | `H29 H.1 \ h14 h.1 \ V32 V.38` | 60с (1 хв) | 1740с (29 хв) | 0.1с | 14с | 32% | 38% |
| **ZS6** *(3 оберти)* | P1 (Р-1) | `H13 H.2 \ h14 h.1 \ V37 V.55` | 120с (2 хв) | 780с (13 хв) | 0.1с | 14с | 37% | 55% |
| **ZS6** | P2 (Р-2) | `H29 H.1 \ h14 h.1 \ V37 V.55` | 60с (1 хв) | 1740с (29 хв) | 0.1с | 14с | 37% | 55% |
| **ZS8** | P1 (Р-1) | `H10 H.2 \ h14 h.1 \ V37 V.55` | 120с (2 хв) | 600с (10 хв) | 0.1с | 14с | 37% | 55% |
| **ZS8** | P2 (Р-2) | `H29 H.1 \ h14 h.1 \ V37 V.55` | 60с (1 хв) | 1740с (29 хв) | 0.1с | 14с | 37% | 55% |

Work Time, Rest Time, Ramp Up і Ramp Down у цій прошивці за замовчуванням
уже збігаються з рядком **ZS4** з таблиці вище (120с/480с для P1,
60с/1740с для P2, 0.1с/14с розгін/спад). **Max Power і Min Power за
замовчуванням (75%/55%) з відео не збігаються** — це або інша модель,
або значення підібрані вручну під конкретний насос (див. примітку ⚙️
вище про калібрування Max/Min Power під реальне зрошення шайб).

### Захист від аварії

Якщо димер активний (`level ≥ 5%`), а датчик струму (`ACS712`) не бачить
струму понад **Fault: Detection Time** секунд — система вважає це аварією
(перевантаження / заклинювання ротора), вимикає насос, чекає
**Fault: Retry Delay** секунд і пробує запустити знову. Лічильник спроб
(`Restart Attempts`) видно у веб-інтерфейсі. Функцію можна вимкнути
перемикачем **Fault Detection: Enabled**.

### Вимірювання струму (RMS)

Через фазове різання димера струм насоса — не гладка синусоїда, а короткі
імпульси. Тому `Pump Current` рахується як справжнє RMS по ~200 прямих
відліках АЦП за ~40мс (≈2 періоди мережі), а не одним "миттєвим" знімком —
це усуває стрибки показань, характерні для наївного одноразового
вимірювання при фазовому регулюванні.

### Світлодіодна індикація (LED, GPIO2)

Один світлодіод на платі показує поточний стан системи без потреби
відкривати веб-інтерфейс. Логіка "тіків" (`g_led_tick`) генерує цикл із
20 тіків по 100мс = **2 секунди на повний цикл малюнка**, і саме на
довжині цього циклу і будуються всі патерни нижче.

| Стан | Що показує LED | Патерн (1 клітинка = 100мс, всього 2с) |
|------|-----------------|------------------------------------------|
| **0 — Stopped** | Погашений постійно | `░░░░░░░░░░░░░░░░░░░░` |
| **1 — Running** | Горить постійно (без миготіння) | `████████████████████` |
| **2 — Resting** | 1 короткий спалах раз на 2с (100мс on / 1900мс off) | `█░░░░░░░░░░░░░░░░░░░` |
| **3 — Fault** | Подвійний спалах раз на 2с (2×200мс on з паузою, потім довга пауза) | `██░░██░░░░░░░░░░░░░░` |

*(`█` = LED увімкнено, `░` = вимкнено; читати зліва направо як 20 послідовних відрізків по 100мс)*

Тобто, дивлячись лише на індикатор, можна на око визначити стан:
- **не блимає і не горить** → насос зупинений (Pump: Enabled = OFF або P3 ще не стартував);
- **горить рівно, без миготіння** → зараз фаза "робота" (Running);
- **одна коротка "цятка" раз на ~2с** → фаза "пауза" між циклами (Resting) — усе штатно, просто чекає;
- **два коротких спалахи підряд раз на ~2с** → аварія (Fault), система чекає перед повторною спробою запуску — варто перевірити насос/трубу, якщо це повторюється (див. лічильник `Restart Attempts`).

> Примітка: у режимі **P3 Constant** LED веде себе як у стані Running
> (горить постійно), доки не станеться аварія — окремого патерна для P3
> немає, бо для контролера це просто різновид "роботи".

## 4. Веб-інтерфейс (`web_server`, порт 80)

Налаштування згруповані на сторінці пристрою:

| Група | Що містить |
|-------|-----------|
| **Pump Control** | режим роботи (P1/P2/P3), вмикання/вимикання насоса |
| **Power Settings** | Max Power / Min Power (%) — спільні для P1 і P2, підбираються дослідно під конкретний септик (див. "Роль циркуляційного насоса" вище) |
| **Status & Monitoring** | поточна фаза, залишок часу (M:SS), потужність, струм, статус роботи |
| **Mode P1 — Standard** | час роботи/паузи/розгону/спаду + текстова хвилинна підказка під кожним параметром |
| **Mode P2 — Economy** | те саме для економ-режиму |
| **Fault & Recovery** | час виявлення аварії, затримка перезапуску, лічильник спроб |

## 5. Встановлення / прошивка

1. Встановіть ESPHome:
   ```bash
   pip install esphome
   ```
2. Відкрийте `pump_controller.yaml` і замініть плейсхолдери на свої реальні
   значення (Wi-Fi, API-ключ, OTA-пароль, пароль fallback AP):

   | Плейсхолдер у файлі | Що підставити |
   |----------------------|----------------|
   | `YOUR_WIFI_SSID` / `YOUR_WIFI_PASSWORD` | ваша основна WiFi-мережа |
   | `YOUR_WIFI_SSID_2` / `YOUR_WIFI_PASSWORD_2` | резервна WiFi-мережа |
   | `YOUR_API_ENCRYPTION_KEY_BASE64` | ключ шифрування Home Assistant API — згенерувати: `python3 -c "import base64,os; print(base64.b64encode(os.urandom(32)).decode())"` |
   | `YOUR_OTA_PASSWORD` | свій пароль для OTA-прошивки |
   | `YOUR_AP_FALLBACK_PASSWORD` | пароль fallback Access Point (з'являється, якщо WiFi недоступний) |

   > Файл самодостатній — жодного окремого `secrets.yaml` не потрібно,
   > але саме тому **не публікуйте його публічно після заповнення реальними
   > значеннями** (це вже не плейсхолдери, а справжні паролі/ключі).
3. Перша прошивка — через USB (потрібен кабель до Wemos D1 mini):
   ```bash
   esphome run pump_controller.yaml
   ```
4. Усі наступні оновлення — по мережі (OTA), без кабелю:
   ```bash
   esphome upload pump_controller.yaml
   ```
5. Веб-інтерфейс пристрою: `http://<ip-адреса-пристрою>/` (за замовчуванням
   у конфігурації задано статичну адресу `192.168.5.123` — за потреби
   змініть `use_address` у `wifi:`).

## 6. Безпека

- Після того, як заповните плейсхолдери реальними Wi-Fi паролями, API-ключем
  та OTA-паролем, **не публікуйте і не комітьте цей `pump_controller.yaml`
  у публічний git/хмару** — на відміну від шаблону з плейсхолдерами, файл
  із заповненими значеннями вже містить справжні секрети.
- Усі роботи з проводкою 220В — тільки знеструмленими руками фахівця,
  через автоматичний вимикач і УЗО.
